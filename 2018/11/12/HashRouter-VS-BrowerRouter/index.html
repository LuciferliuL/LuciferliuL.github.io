HashRouter VS BrowerRouter

Posted on 2018-11-12 | Visitors: ℃

RR4 推荐使用BrowerRouter

原因很简单：BrowerRouter 使用H5的History API，浏览器提供接口来修改浏览器历史记录，Hash是通过改变地址后面的hash来改变历史记录。History API 提供pushState() 和 replaceState() 方法来增加或者替换历史记录，而hash没有相应的方法。还有一个原因是hash部分不会被浏览器发送到服务端，不论是#XXX 还是#AAA ,服务端统一知道请求了index.html，不能获取细节。而History API可以提供，需要服务器支持。

问题：使用BrowerRouter，刷新显示404的原因分析

SPA router跳转是前端js的改变location，并没有刷新页面访问后台。点击刷新，首先访问的是后台地址，然后加载前端，返回404就是后台并没有返回这个route需要的html。

解决：配置后台服务器，做通用路径处理，全部返回index.html

对于使用nginx服务器，可以加上try_files配置

location / { .... try_files $url /index.html }

PS:build以后的路径也需要在webpack里面修改成绝对路径

在前端路由模式还在探索阶段，强烈建议继续使用HashRouter
