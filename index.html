HashRouter VS BrowerRouter

Posted on 2018-11-12 | Visitors: ℃

RR4 推荐使用BrowerRouter

原因很简单：BrowerRouter 使用H5的History API，浏览器提供接口来修改浏览器历史记录，Hash是通过改变地址后面的hash来改变历史记录。History API 提供pushState() 和 replaceState() 方法来增加或者替换历史记录，而hash没有相应的方法。还有一个原因是hash部分不会被浏览器发送到服务端，不论是#XXX 还是#AAA ,服务端统一知道请求了index.html，不能获取细节。而History API可以提供，需要服务器支持。

问题：使用BrowerRouter，刷新显示404的原因分析

SPA router跳转是前端js的改变location，并没有刷新页面访问后台。点击刷新，首先访问的是后台地址，然后加载前端，返回404就是后台并没有返回这个route需要的html。

解决：配置后台服务器，做通用路径处理，全部返回index.html

对于使用nginx服务器，可以加上try_files配置

location / { .... try_files $url /index.html }

PS:build以后的路径也需要在webpack里面修改成绝对路径

在前端路由模式还在探索阶段，强烈建议继续使用HashRouter

Map对象使用与数组对象去重

Map 对象保存键值对。任何值(对象或者原始值) 都可以作为一个键或一个值

语法 new Map([iterable])
参数：可以是数组也可以是对象，每个元素都以键值对的形式添加到Map

Map 实质

Map对象根据传入的元素进行迭代按顺序形成[key,value]的数组
key equality 比较是基于 SameValueZero 算法：NAN 与 NAN是相等的(NaN !== Nan),其余的都是根据===计算

Map 属性

Map.length 值为0

Map 实例

Map.prototype.clear()
移除Map对象的所有键/值对 。
Map.prototype.delete(key)
如果 Map 对象中存在该元素，则移除它并返回 true；否则如果该元素不存在则返回 false
Map.prototype.entries()
返回一个新的 Iterator 对象，它按插入顺序包含了Map对象中每个元素的 [key, value] 数组。
Map.prototype.forEach(callbackFn[, thisArg])
按插入顺序，为 Map对象里的每一键值对调用一次callbackFn函数。如果为forEach提供了thisArg，它将在每次回调中作为this值。
Map.prototype.get(key)
返回键对应的值，如果不存在，则返回undefined。
Map.prototype.has(key)
返回一个布尔值，表示Map实例是否包含键对应的值。
Map.prototype.keys()
返回一个新的 Iterator对象， 它按插入顺序包含了Map对象中每个元素的键 。
Map.prototype.set(key, value)
设置Map对象中键的值。返回该Map对象。
Map.prototype.values()
返回一个新的Iterator对象，它按插入顺序包含了Map对象中每个元素的值 。
Map.prototype@@iterator
返回一个新的Iterator对象，它按插入顺序包含了Map对象中每个元素的 [key, value] 数组

使用Map实现一句话数组对象去重

解读 :

  1. 接受一个数组
  2. 创建新的Map对象
  3. 形参数组筛选
  4. res.has(protor) 是否拥有该属性 true返回false筛选为重复，false返回true
  5. 进入false的Obj 给他一个属性用来下次判断是否重复
  6. 返回筛选出来的数组
